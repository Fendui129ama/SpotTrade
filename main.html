Help

Buy order: You lock quote (ETH). Required value = size × price / 10000. When matched, you receive the base (size in wei) and the taker receives quote minus fee.
Sell order: You lock base (size in wei). Required value = size. When matched, you receive quote minus fee and the taker receives the base.

Price tick is in basis-point style (e.g. 10000 = 1:1, 9500 = 0.95).

Expiry block: set to 0 for no expiry; otherwise the order becomes invalid after that block.

    Glossary
    
        
    Maker: Who places the limit order. Taker: Who gets filled against the order (when matcher runs matchOrder). Price tick: Basis-point style price (10000 = 1.0). Notional: size × price tick / 10000. Matcher: Trusted address that can call matchOrder. Treasury / Fee vault: Immutable addresses that receive half of fees each.
    
      
    
      
    
        
    Quick reference — required ETH
    
        
    Place buy order: required = (size × priceTick) / 10000 (in wei). Place sell order: required = size (in wei). Match buy order: matcher sends fillWei (base). Match sell order: matcher sends notional = (fillWei × priceTick) / 10000.
    
        
    Order size must be between min and max order size (see stats bar). Price tick must be at least 1. Each maker can have up to 256 open orders.
    
        
    Cancel or expiry refunds the maker: for buy orders the unused notional; for sell orders the unfilled size. Keeper can expire orders past their expiry block via the contract.
    
      
    
      
    
        
    About TradeMatch
    
        
    TradeMatch is a limit order matching service on EVM chains. Makers place buy or sell orders with a price tick and size; the matcher fills orders on behalf of takers. Fees are split between treasury and fee vault. Orders can have an optional expiry block. Use SpotTrade to place, cancel, lookup, and (as matcher) match orders.
